## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Replace the bridge fork with a narrow qq-owned delegate runtime that implements only qq's settled production contract. The runtime launches one canonical role into one assigned same-Repository worktree with fresh role context and an exact completion schema; owns asynchronous lifecycle evidence and completion notification; supports fail-closed status, stop, wait, and contract-preserving resume; and reuses `bin/qq-dispatch` for Landstrip confinement, hard timeout, process-tree cleanup, runtime isolation, and observation.

Start by extracting the bridge's black-box contract suite so the same Checks run against both implementations. Prefer Pi's documented subprocess/JSON or RPC surfaces and the official subagent example over private Pi internals. Keep orchestration, role policy, and lifecycle in qq while Landstrip remains a replaceable driver. Preserve native Pi session JSONL as the sole agent-content observation seam, and adapt observation assembly explicitly rather than emulating undocumented pi-subagents internals.

Non-goals: chains; generic or dynamic fan-out; generic agent discovery; schedulers; best-of-N; sharing; watchdogs; model fallback/routing; acceptance inference; a new TUI; confidentiality or hostile-code containment; changing delegate network policy; and resolving the separate observer Change-attribution design.

## Decision ledger

- `decision-12` — qq owns this narrow destination runtime and retires the fork only after contract equivalence is proven.
- `decision-8` — Landstrip remains the exact-version open-egress drift-net beneath qq-owned supervision; this runtime adds no broader security claim.
- `decision-10` — lifecycle metadata may be live, but agent content is observed only from persisted Pi session JSONL.
- `T-152` / `doc-88` — canonical role occupancy and execution-profile policy are qq-owned, fail closed, and cannot be overridden by same-name project definitions or delegate-package routing.

The attached plan settles the product boundary, not every implementation detail. Realign before adding a new public workflow, lifecycle authority, observation seam, or security boundary.

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 One qq-owned tool launches exactly one trusted canonical role with fresh context, one assigned worktree, and one exact completion schema; unknown, conflicting, or untrusted role occupancy fails before child launch.
- [ ] #2 The runtime persists an immutable launch descriptor and lifecycle artifacts sufficient for deterministic status, completion notification, stop, wait, crash diagnosis, and resume without contract drift.
- [ ] #3 Hard timeout, termination forwarding, descendant cleanup, worktree validation, Landstrip policy identity, scratch isolation, and session preservation remain enforced through qq-dispatch and fresh Checks.
- [ ] #4 Recovered child tool failures followed by valid terminal structured completion succeed; missing, malformed, invalid, or failed structured completion and unrecovered terminal errors fail.
- [ ] #5 Resume preserves the original role, cwd, session, schema, output, timeout, and context contract and refuses malformed, cross-run, stopped, ambiguous, or foreign descriptors.
- [ ] #6 Observer assembly consumes the qq-owned lifecycle/session mapping without adding a live agent-content capture path and without losing canonical Pi session evidence.
- [ ] #7 code-review and delegate-batch pass their existing workflow and completion-envelope Checks against the qq runtime before the fork package, its installation pin, and pi-subagents-specific production coupling are removed.
- [ ] #8 The resulting source and public tool surface exclude the listed generic orchestration non-goals; any required expansion is realigned before implementation.
